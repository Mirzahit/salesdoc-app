# SalesDoc Telegram Bot — пошаговая настройка

Один раз — потом всё крутится само.

## Что получишь в итоге

- Утром в 08:00 каждый оператор (Айдос/Акбар/Самат) получает в Telegram свою сводку: «X клиентов в работе, Y просрочено».
- При появлении новой карточки на канбане — пинг тому оператору, кому она досталась (или всем, если без оператора).
- Если карточка торчит на этапе дольше лимита SLA — алерт оператору. Молчит 24 часа — эскалация CEO.
- Команды `/today`, `/clients`, `/c название` — оператор управляет клиентами из телефона.

---

## Шаг 1 — Создать бота в Telegram

1. Открой в Telegram `@BotFather`
2. `/newbot`
3. Имя: `SalesDoc Operators` (или любое)
4. Username: `salesdoc_ops_bot` (или любой уникальный, должен заканчиваться на `_bot`)
5. Скопируй **TOKEN** который он пришлёт. Выглядит как `1234567890:AAH-RsXh...`
6. (Опционально) `/setdescription` — описание; `/setcommands` — для подсказок:
   ```
   menu - Главное меню
   today - Задачи на сегодня
   clients - Мои клиенты
   c - Поиск клиента (например: /c karat)
   ```

## Шаг 2 — Развернуть Apps Script

1. Открой [script.google.com](https://script.google.com) — создай **новый проект**, назови `SalesDoc Bot`
2. В файле `Код.gs` удали всё и вставь содержимое `bot.gs` из этого репозитория
3. Слева ⚙️ **Project Settings** → блок **Script Properties** → **Add script property**:
   - `BOT_TOKEN` = токен из шага 1
   - `SPREADSHEET_ID` = `1lXBeS-TTH2SihtV_6Ki4O_nnuNaC9xza2esw0sZnZKc` (тот же что у канбана)
4. Слева **Editor** → выбери функцию `initSheets` → **Run**. Дай разрешения. Проверь — в таблице должны появиться листы `Operators`, `Bot Events`, `Bot SLA`.

## Шаг 3 — Опубликовать как Web App

1. Справа сверху **Deploy** → **New deployment**
2. **Select type** (шестерёнка) → **Web app**
3. Параметры:
   - **Execute as**: `Me`
   - **Who has access**: `Anyone` ← обязательно, иначе Telegram не сможет дёрнуть
4. **Deploy** → скопируй **Web app URL**. Выглядит так:
   ```
   https://script.google.com/macros/s/AKfycb.../exec
   ```

## Шаг 4 — Привязать Telegram к Web App (webhook)

Открой в браузере (замени `<TOKEN>` и `<WEB_APP_URL>`):

```
https://api.telegram.org/bot<TOKEN>/setWebhook?url=<WEB_APP_URL>
```

Должно вернуть `{"ok":true,"result":true,"description":"Webhook was set"}`.

Проверка что вебхук установлен:
```
https://api.telegram.org/bot<TOKEN>/getWebhookInfo
```

## Шаг 5 — Триггеры (чтобы бот сам работал по времени)

В Apps Script слева ⏰ **Triggers** → **Add Trigger** для каждого:

| Function       | Time-driven  | Frequency       |
|----------------|--------------|-----------------|
| `dailyMorning` | Day timer    | 8am–9am         |
| `slaCheck`     | Hour timer   | Every hour      |
| `eventPusher`  | Minutes timer| Every minute    |

## Шаг 6 — Привязать CEO (тебя)

1. В Google Sheet открой лист `Operators` — добавь строку:
   ```
   Мирзахит | (пусто) | CEO-START | ceo | TRUE | Asia/Almaty
   ```
2. Открой своего бота в Telegram, напиши:
   ```
   /start CEO-START
   ```
3. Бот ответит «Готово, Мирзахит!». В Sheets во второй колонке появится твой `telegram_id`.

## Шаг 7 — Привязать операторов

Для каждого (Айдос/Акбар/Самат):

1. В Apps Script Editor → выбери функцию `generateInviteCode` → **Run** один раз для генерации, но проще руками в Sheets:
2. В листе `Operators` в колонке `invite_code` для строки Айдоса впиши, например, `AIDOS-A47K2`
3. Передай Айдосу: «Открой @salesdoc_ops_bot, напиши `/start AIDOS-A47K2`»
4. После того как он напишет — в Sheets автоматически появится его `telegram_id`. Поле `invite_code` обнулится (код одноразовый).

То же для Акбара и Самата с уникальными кодами.

## Шаг 8 — Настройка SLA-лимитов

В листе `Bot SLA` редактируй колонку `limit_days`. Дефолты:

| stage                   | limit_days |
|-------------------------|------------|
| В очереди               | 3          |
| Взят в работу           | 7          |
| Настройка сервера       | 5          |
| Обучение полевых        | 7          |
| Обучения офисных        | 5          |
| Обучение Руководства    | 3          |
| Активация               | 2          |

Меняй прямо в Sheets — бот подхватит на следующем `slaCheck`.

## Если что-то не работает

- **Webhook не ставится** — проверь что **Execute as: Me** и **Who has access: Anyone** при деплое Web App.
- **Бот не отвечает** — открой Apps Script → **Executions** (слева, ⚡). Видны ошибки. Чаще всего — пропущен Script Property или нет прав на Sheets.
- **Не приходит утренняя сводка** — проверь триггеры (шаг 5). Триггер `dailyMorning` должен быть `Day timer` 8–9am.
- **Бот не находит клиента** — попробуй точнее, минимум 2 символа. Имя клиента сравнивается без учёта регистра.

## Логи

Все события (новые сообщения, алерты, ошибки) пишутся в Sheet → лист `Bot Events`. Удобно проверять что бот делает в реальном времени.

## Следующие шаги (когда захочешь)

- Подключение бухгалтера и менеджеров (просто добавь строки в `Operators` с `role=accountant` или `manager`)
- Голосовые заметки → текст → в карточку клиента (нужен Whisper API, ~$5/мес)
- Тёплые/холодные клиенты — добавить кнопки в карточку
- Шорткаты «Закрыл звонок: …» — авто-создание задачи на следующий контакт
