# Раздел «Внедрение» — реструктуризация в табы (этап 1)

**Дата:** 2026-05-06
**Файл:** `index.html` (монолитный SPA)
**Контекст:** раздел `view-kanban` в SalesDoc Dashboard

## Цель

Разбить раздел «Внедрение» на 3 равноправные подстраницы (табы) и убрать активированных клиентов с канбан-доски — они засоряют рабочее пространство. Архив проектируется так, чтобы в будущем стать точкой передачи в отдельный канбан «Производство и Поддержка» (вне этого этапа).

## Что НЕ входит в этап 1

- Перегруппировка CRM-полей в карточке клиента (этап 2, отдельный spec).
- Изменения дизайна (цвета колонок, типографика, отступы).
- Сам канбан «Производство и Поддержка».
- Любые правки логики операторов/KPI (есть в других местах).

## Бэклог (этапы 3+, в этом spec не реализуем)

Идеи на будущее, чтобы не потерять:

- **Документы на карточке клиента.** Вкладка «Документы»: договор, АВР, ТЗ, прайс — ссылки или загрузка в Google Drive. Сейчас разбросано по чатам.
- **SLA по этапам, а не общий «7 дней».** Норма aging своя для каждой колонки (например, «Тест» 1–2 дня, «Обучение» до 10), чтобы видеть реальные задержки.
- **Чеклист по типу внедрения** (`implType: Стандарт / Расширенное / VIP`). Сейчас чеклист единый — стоит сделать вариативный.
- **Автоматические задачи при смене этапа.** Карточка в «Обучение» → автоматом ставится задача «назначить дату обучения» нужного типа.
- **Тайм-трекинг** по этапам — сколько реально часов потрачено на каждом этапе.
- **Score сложности внедрения** — индекс по типу/сегменту/размеру для прогноза сроков.

## Архитектура

### Шапка раздела (постоянная над всеми табами)

```
[Доска][Архив][Отчёт]
```

- Табы рендерятся внутри `view-kanban` как тонкая полоса сверху.
- Активный таб — обычная подсветка (используем существующие стили `kb-filter-btn` или новый класс `kb-tab`).
- Доска — таб по умолчанию при входе в раздел.
- Текущий топбар канбана (`kb-topbar` со счётчиком, поиском, фильтрами, «Отчёт»/«Настройки»/«Обновить») рендерится только когда активен таб «Доска».
- Общего селектора периода в шапке нет (если понадобится — добавляется локально в Архив/Отчёт).

### Три внутренних view

`view-kanban` становится контейнером с тремя секциями:

- `kb-tab-board` (старое содержимое — топбар + `kb-board` + scroll dots)
- `kb-tab-archive` (новое)
- `kb-tab-report` (содержимое из текущего `view-kb-report`, перенесённое внутрь)

Переключение табов — обычный JS: показываем нужный блок, прячем остальные.

## 1. Доска

### Что меняется

- Карточки с заполненной `activationDate` (через `clientActivMap` или `saved.activDate`) **не рендерятся на доске**. Они теперь в Архиве.
- Логика построения `kbCards` дополняется флагом `card._archived` — карточка остаётся в массиве, но `kbRenderBoard()` фильтрует её.
- Колонка-сегмент «Активирован» (если рендерилась как отдельная колонка для этих карточек) — убирается с доски. Дефолтный `kbColumns` остаётся: `['В очереди','Интеграция','Обучение','Настройка','Тест','Запущен']`.
- Сегмент-фильтр «Активированные» в выпадающей панели поиска (`kbsp-segments`) **удаляется** — теперь это отдельный таб.

### Что остаётся

- Топбар, поиск, фильтры (оператор/менеджер/этап), drag-and-drop, чеклист, WIP-лимит, aging, цвета колонок, fullscreen-карточка клиента.

### Триггер архива

Карточка считается архивной если `clientActivMap[clientKey]` существует **или** `saved.activDate` проставлен. Никаких новых полей в Sheets — данные те же.

## 2. Архив (новая подстраница)

### Что показывает

KPI-полоска сверху:
- «Активировано всего: X»
- «Среднее время от старта до активации: Y дней» (по `kbLog` — первый move карточки → дата активации)
- «За месяц: Z»

Таблица:

| Клиент | Дата активации | Оператор | Менеджер | Тип | Сумма | Дней на доске | Действие |

- Сортировка по умолчанию: дата активации, по убыванию.
- Поиск: переиспользуем существующий `kb-search-input` (когда активен таб «Архив», вводимая строка фильтрует таблицу архива, не доску).
- Фильтры: оператор, менеджер, период активации (локальный селектор «Месяц / Квартал / Год / Всё» — по умолчанию «Всё»).
- Клик по строке — открывает ту же fullscreen-карточку клиента (`kbOpenCard`), но в шапке карточки бейдж «Активирован» + действия по этапам отключены.

### Действия

- **«Вернуть на доску»** — записывает override в `kb_cards_local` (localStorage), который игнорирует `activationDate` для этой карточки. Карточка возвращается в колонку, на которой была до активации (берём из `kbLog` последний `to` ≠ «Активирован», иначе — последняя колонка пайплайна).
- При следующем сохранении/синхронизации с Карточками override продолжает действовать, пока пользователь снова не активирует клиента или явно не уберёт override.

### Хранение данных

**Новых таблиц/листов в Google Sheets не создаём.** Архив — это view над `kbCards` (фильтр `card._archived === true`). Возврат — флаг в localStorage, без записи в Доходы или Карточки.

## 3. Отчёт (перенос fullscreen → подстраница)

- Текущий блок `#view-kb-report` (KPI / Воронка / Aging / Сводка по этапам / Сводка по операторам) переезжает внутрь `view-kanban` как `kb-tab-report`.
- Кнопка «Отчёт» в топбаре доски (`kbOpenReport()`) удаляется — её роль играет таб.
- Метрики считаются по тем же `kbCards` + `kbLog`. Архивные карточки **включаются** в воронку и aging (как и сейчас) — отчёт показывает картину целиком, не «только активные».
- Функция `kbCloseReport()` тоже удаляется — она больше не нужна.

---

## Связь с будущим канбаном «Производство и Поддержка»

Архив проектируется как **«готов к передаче»**, не финальный тупик:

- Структура карточки (`kbCards[i]`) не меняется — она и так переживёт переезд.
- Действие «Вернуть на доску» в будущем дополнится действием «Передать в производство», которое перенесёт карточку в новый канбан.
- В Архиве не делаем функционала, который дублирует будущий «Производство/Поддержка» (тикеты, доработки, чеклист сопровождения и т.д.).

## Тестирование

Ручная проверка (UI-фича в монолитном SPA, автотестов нет):

- Заходим в раздел «Внедрение» → видим табы [Доска][Архив][Отчёт], открыта Доска.
- На доске нет колонки «Активирован», нет сегмента-фильтра «Активированные», карточек активированных клиентов на доске не видно.
- В Архиве: KPI-полоска, таблица с активированными клиентами, сортировка по дате активации.
- Клик «Вернуть на доску» — карточка возвращается на доску, при перезагрузке страницы остаётся на доске.
- Активация клиента (запись `activationDate` в Доходы) → после `kbLoad()` карточка пропадает с доски, появляется в Архиве.
- Таб «Отчёт» показывает то же что раньше открывал fullscreen-вью.
- Поиск/фильтры на Доске не аффектят Архив и наоборот.
- Права: `view_kanban` остаётся единым флагом, все 3 таба под ним.
