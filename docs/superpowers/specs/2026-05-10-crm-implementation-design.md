# CRM для операторов техподдержки внедрения — Phase 1 (расширение карточки + календарь)

**Дата:** 2026-05-10
**Версии:** v224 → v229 (примерно)
**Цель Phase 1:** превратить карточку клиента в полноценный CRM-паспорт + добавить календарь оператора. Журнал звонков/чатов НЕ делаем (нет интеграции с телефонией/мессенджерами).

> Это первый из трёх модулей "Тех.поддержка". Phase 2 — действующие клиенты после активации, Phase 3 — сводный дашборд тех.поддержки. Они будут отдельными спеками.

## Контекст и мотивация

Текущий канбан Внедрения (`view-kanban` в `index.html`) — это пайплайн новых клиентов от оплаты до активации. Карточка содержит: один контакт, плоский чеклист на 6 пунктов, поле АВР, задачи в JSON. Этого недостаточно для качественного заведения клиента FMCG-дистрибьютора, где параллельно работают директор, IT-админ, бухгалтер и супервайзеры; на каждом этапе свои подзадачи; есть договор+ТЗ+скрины+видео; нужно планировать обучения и выезды.

Phase 1 закрывает 5 пробелов:
1. Несколько контактов клиента (директор/IT/бухгалтер/супервайзеры)
2. Детальный чеклист с разбивкой по этапам
3. Файлы клиента (договор, ТЗ, скрины, видео — не только АВР)
4. Расписание обучений
5. Календарь оператора (отдельная страница) с фильтрами

## Скоп

**Включено:**
- Расширение карточки клиента (`view-kb-card`) — 5 табов: Чек-листы / Контакты / Файлы / Календарь / История
- Pipeline-визуализация в шапке карточки (горизонтальная воронка с пройденными ✓ / текущим ●)
- Детальные чеклисты по этапам (вместо одного плоского)
- 3 новые таблицы Google Sheets как relation-модель данных:
  - `Контакты` — несколько контактов на клиента
  - `Файлы` — файлы клиента
  - `События` — расписание обучений / встреч / дедлайнов
- Новый раздел `Календарь` в sidebar — Apple Calendar-стиль (День/Неделя/Месяц + фильтр оператор/тип)
- Мини-календарь в табе карточки клиента (только события этого клиента)
- Новые Apps Script endpoints для CRUD по 3 таблицам
- Дизайн на токенах Emerald Depth + Navy Mirage (как утверждено в `mockup-crm.html`)

**Не включено (Phase 2/3 или out-of-scope):**
- Журнал звонков/чатов (нет интеграции с телефонией)
- Управление действующими клиентами после активации (Phase 2)
- Сводный дашборд тех.поддержки руководителю (Phase 3)
- NPS/оценка клиента (out-of-scope для Phase 1)
- Push-уведомления о дедлайнах (можем добавить позже)
- Интеграция с Google Calendar внешним

### Как работают файлы

Загрузка через Apps Script POST с base64 (как АВР сейчас, line ~17500-17700). Два режима:
1. **Маленькие файлы (< 5 МБ)** — base64 в POST → Apps Script → `DriveApp.createFile` → возврат `drive_url` и `drive_file_id`
2. **Большие файлы (≥ 5 МБ)** — оператор вставляет sharing-link с своего Google Drive вручную (поле "Ссылка"), без upload

Фронт показывает progress-bar для маленьких, для больших просто принимает URL.

### Как работает календарь / задачи (TickTick-стиль)

**Концепция:** не просто календарь, а **система задач** — каждое событие это todo-item с checkbox, приоритетом и связью с клиентом. Календарная сетка — вторичный способ просмотра.

**Источник данных:** Sheet `События` (см. структуру выше). Поля расширены под TickTick:
- `priority` (col 13): 1=red / 2=orange / 3=yellow / 4=blue / 0=none
- `recurring` (col 14): пусто / `daily` / `weekly` / `monthly` / RRULE-формат
- `parent_id` (col 15): для подзадач (опц., Phase 1.5)

**Состояние в JS:** глобальный массив `crmEvents`.

**Layout страницы (3 колонки, как mockup-crm.html `tt-page`):**
- **Левый sidebar (240px)** — Smart Lists + фильтры:
  - Списки: Сегодня / Завтра / 7 дней / **Просрочено** (с red-счётчиком) / Без даты / Выполненные
  - Операторы: Айдос/Акбар/Самат с цветными точками + счётчиком
  - Типы: Обучения/Встречи/Дедлайны/Прочее
- **Центральная панель (flex)** — agenda-list или calendar-grid (переключатель "Список / Календарь")
  - **Список (default):** sticky-заголовки дней `12 Понедельник, мая` + строки задач (checkbox + title + meta: время · тег · клиент · оператор)
  - **Календарь:** месячная сетка 6×7 (Apple-style), события как chips
- **Правый sidebar (320px)** — мини-календарь Месяц + детали выбранной задачи

**Quick-add (внизу центральной панели):**
- Inline input "+Добавить задачу…"
- Парсинг естественного языка:
  - «Завтра 10:00 встреча с ТОО El-product» → date=tomorrow, time=10:00, type=meeting, client=match
  - «Через неделю обучение» → date=+7d, type=training
  - «Сегодня !1 дедлайн» → date=today, priority=1
  - Распознавание клиентов: substring-match на `kbCards[].client`
  - Распознавание операторов: substring-match на `KB_OP_COLORS`

**Drag-and-drop:**
- Перенос задачи между днями в agenda-list (HTML5 drag, обновляет `date`)
- Перенос между Smart-lists (например drag в "Завтра" → date=tomorrow)
- Перенос на календарной сетке между ячейками-датами

**Приоритеты (P1-P4):**
- Цветной кружок-checkbox (red/orange/yellow/blue)
- Сортировка внутри дня по priority desc → time asc

**Повторяющиеся задачи:**
- При создании опция "Повторять": ежедн / еженед / ежемес / по будням / каждые N дней
- Шаблон сохраняется один раз; экземпляры разворачиваются view-time (не материализуются в Sheet)
- Когда экземпляр отмечается как done — создаётся override-запись (status=done на конкретную дату)

**Calendar grid в карточке клиента (мини):** только список ближайших 5 событий этого клиента (агenda-стиль).

**По умолчанию:** Smart-list "Сегодня", фильтр оператор=текущий пользователь (operator) или "Все" (CEO).

**Что НЕ делаем в Phase 1 (можно в 1.5/2):**
- Pomodoro-таймер
- Habit tracker
- Subtasks (parent_id поле зарезервировано, но UI позже)
- Push-уведомления (можно через Web Notifications API позже)
- Напоминания за 15/30/60 мин до события

### Backend (Apps Script)

Существующий Web App URL поддерживает actions: `updateCard`, `updateChecklist`, `addComment`, `uploadAct`. Расширяем:

- `crmContactsList(client)` → возврат всех контактов клиента
- `crmContactsUpsert({client, name, role, phone, whatsapp, telegram, email, notes})` → создать/обновить
- `crmContactsDelete(id)`
- `crmFilesList(client)` → все файлы
- `crmFilesUpload({client, type, filename, mime, base64, uploaded_by})` → DriveApp.createFile, вернуть URL+ID
- `crmFilesDelete(id)` → DriveApp.removeFile
- `crmEventsList({operator, dateFrom, dateTo})` → события в диапазоне
- `crmEventsUpsert({...})` / `crmEventsDelete(id)`

Apps Script читает/пишет в три новые вкладки. При первом запуске — авто-создание вкладок если их нет (одноразовая миграция).

## Data flow

### Загрузка карточки клиента
1. `kbOpenCardPage(card)` вызывает 3 fetch'а параллельно: `crmContactsList`, `crmFilesList`, `crmEventsList(client=...)`
2. Каждый кладёт результат в кэш (`crmContactsByClient[client]`)
3. После всех — рендерит табы

### Загрузка календаря
1. `showView('calendar')` вызывает `crmEventsList({operator: currentFilter, dateFrom, dateTo})`
2. Кладёт в `crmEvents`
3. Рендерит сетку (месяц/неделя/день) + фильтры

### Создание события
1. Клик "+ Создать" в карточке клиента или на пустом слоте календаря
2. Открывается модалка с полями (тип/название/дата/время/место/заметки)
3. POST `crmEventsUpsert`, по success — push в `crmEvents` локально + перерендер

## Error handling

- Все fetch'и — с try/catch и toast при ошибке
- При offline (нет ответа от Apps Script) — режим read-only, показать banner "Изменения не сохранятся"
- Файлы > 25 МБ — отклоняем с понятным сообщением
- Если Sheet вкладки не существуют — показать инструкцию "Свяжитесь с админом"
- Конфликты при одновременном редактировании — last-write-wins (без CRDT, у нас 3 оператора)

## UI / Design

**Палитра пересмотрена:** уход от Emerald Depth к **iOS systemColors** — пользователь хочет «как на iPhone» (Apple Mail/Notes/Settings). Это **глобальная** правка программы, не только новый раздел.

- `--accent: #0A84FF` (iOS systemBlue, dark) / `#007AFF` (light) — основной акцент: tap-actions, links, active states, кнопки-primary, активный пункт sidebar
- `--accent-bright: #5AC8FA` — accent для текста (cyan-light, лучше читается на чёрном)
- `--done: #30D158` (iOS systemGreen) — **только статус «выполнено / готово»** (галочки, прогресс-бары, бейдж done). Это не основной акцент.
- `--p1: #FF453A` (red), `--p2: #FF9F0A` (orange), `--p3: #FFD60A` (yellow), `--p4: #0A84FF` (blue) — приоритеты задач (как TickTick P1-P4)
- `--indigo: #5E5CE0`, `--orange: #FF9F0A`, `--red: #FF453A` — для фильтров операторов и типов событий

**Light-вариант:**
- `--accent: #007AFF`, `--accent-bright: #0A84FF`
- `--done: #34C759`
- Фон #F2F2F7, карточки белые

Радиусы 14/18/24, soft shadows iOS, Inter (font-feature `cv11` для тонкой типографики).

Эталон визуала — `mockup-crm.html` (одобрен пользователем после правок: убраны звонки/чат, перекраска emerald → systemBlue).

**Phase 0 (предварительный шаг перед CRM-функциями):** перекрасить существующую программу с Emerald Depth (#28623A) на systemBlue (#0A84FF). ## Тестирование

Проект — vanilla SPA, автотестов нет. Каждый шаг проверяется в браузере (puppeteer-core снимок раздела до/после) с тестовым аккаунтом `office@salesdoc.io` (admin):

1. Открыть Внедрение → клик карточки → новые табы видны → переключаются → данные грузятся
2. Добавить контакт → виден в списке → перезагрузка → сохранён
3. Загрузить маленький файл → виден в списке → клик открывает Drive
4. Создать событие → видно в табе Календарь карточки → видно на странице Календарь
5. Чеклист этапа 100% → кнопка "Активировать" доступна; иначе — заблокирована
6. Sidebar → Календарь → события за неделю отображаются → фильтр оператор работает
