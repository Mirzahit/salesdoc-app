# Рейтинг объявлений (Ad Effectiveness Ranking) — дизайн

Дата: 2026-06-02
Раздел: Маркетинг (salesdoc-app, `index.html` + `api/meta-ads.js`)

## Цель

CEO запускает несколько объявлений (видео, баннеры). Нужно видеть, **какое объявление/креатив приносит больше лидов** и по какой цене, чтобы отключать неэффективные. Действие — отключение — делается вручную в Meta Ads Manager (дашборд только показывает рейтинг, никакой записи в Meta).

## Объём (scope)

- Read-only рейтинг всех объявлений рекламного аккаунта за выбранный период.
- Группировка по креативу (одно видео/баннер = одна строка), с переключателем на «каждое объявление отдельно».
- Метрики: лиды (главная), цена лида, расход, CTR, показы; тип креатива (видео/баннер); статус.
- Подсветка кандидатов на отключение (расход есть, лидов нет / цена лида сильно выше средней).
- Прямая ссылка на объявление в Meta Ads Manager.
- Страны KZ/KG (как везде через `currentCountry`).

Вне объёма: запись в Meta (пауза/правка), привязка к выручке/ROI (нет поля источника у клиентов — отдельный большой проект через amoCRM utm).

## Backend — новый endpoint в `api/meta-ads.js`

`GET /api/meta-ads?endpoint=all_ads&period=<P>&country=<KZ|KG>[&active_only=1]`

Запрос к Meta:
```
/{ACCOUNT}/ads?fields=id,name,effective_status,
  campaign{name},adset{name},
  creative{thumbnail_url,object_type,video_id,image_hash},
  insights.date_preset(PERIOD){spend,impressions,clicks,inline_link_clicks,ctr,actions,cost_per_action_type}
&limit=200
```
- Пагинация: следовать `paging.next` до кап ~500 объявлений (защита от бесконечного цикла).
- Лиды на объявление: переиспользовать существующую `summarizeLeads(actions, cost_per_action_type)`.
- Кэш 10 мин (существующий механизм `cacheGet/cacheSet`).
- Тип креатива: `video_id` есть → `видео`, иначе → `баннер` (по `image_hash`/`object_type`).
- В ответ добавить `account_id` (act_-id, без токена) — фронт строит ссылки на Ads Manager.

Формат ответа:
```json
{
  "period": "...", "country": "KZ", "account_id": "act_...",
  "ads": [{
    "id","name","effective_status",
    "campaign_name","adset_name",
    "creative": {"thumbnail_url","type":"видео|баннер","video_id","image_hash"},
    "spend","impressions","clicks","ctr",
    "leads","cost_per_lead"
  }]
}
```
`cost_per_lead` = `spend/leads` (если leads>0, иначе null).

## Frontend — экран `view-mkt-ads`

- Новая карточка в `view-marketing` (рядом с Таргет/Instagram/Telegram/LTV) → `mktOpenAdsPage()` (паттерн как `mktOpenTargetPage`, класс `sd-overlay-screen`, учитывает фикс z-index v576).
- Шапка: страна (KZ/KG), период (last_7d/last_30d/this_month/last_month и т.д.), чекбокс «Только активные» (по умолчанию вкл).
- Сводка сверху: суммарный расход, всего лидов, средняя цена лида, лучшее объявление, худшее.
- Контролы: сортировка [Лиды (по умолч.) / Цена лида / Расход]; группировка [по креативу (по умолч.) / каждое отдельно].
- Список строк: превью | название + кампания/группа + бейдж тип | Лиды | Цена лида | Расход | CTR | статус | «Meta ↗».
- Группировка по креативу — на клиенте: ключ `video_id || image_hash || thumbnail_url`, суммируем лиды/расход/показы/клики, цена лида и CTR пересчитываются, показываем «в N группах».
- Подсветка: расход>0 и лидов 0 → красный флаг «кандидат убрать»; цена лида заметно выше средней → жёлтый.

### Макет
```
┌ Рейтинг объявлений            [KZ ▾] [30 дней ▾] [Только активные ✓] ┐
│ Сводка: расход 1.2М · лидов 340 · ср. цена лида 3 500 ₸             │
│ Лучшее: «Видео-кейс А» (120 лидов)   Худшее: «Баннер X» (0 лидов)   │
│ Сорт: [Лиды ▾] [Цена лида] [Расход]      Группировать: [по креативу▾]│
│ [превью] Видео-кейс А   видео · кампания «Лидген»                   │
│         Лиды 120 · Цена 1 800 ₸ · Расход 216К · CTR 3.1%   [Meta ↗]│
│ [превью] Баннер X   баннер — расход 80К, 0 лидов  ⚠ кандидат убрать │
└──────────────────────────────────────────────────────────────────┘
```

## Крайние случаи

- Объявление без показов за период → «нет показов», в рейтинге внизу.
- 0 лидов при расходе → красный флаг.
- Нет токена/ошибка Meta → понятная заглушка (как на странице Таргет).
- Много объявлений → пагинация с капом, в сводке отметить если упёрлись в кап.
